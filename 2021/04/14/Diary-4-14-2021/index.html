雪国

《雪国》读完了。不过还是没有读懂作者写这篇小说的意义在哪里 :< 我甚至一直怀疑叶子一直是个疯子，一直幻想自己弟弟还活着，可周围的人都在瞒着她，不太理解为什么最后叶子从火场跳出来。再看一遍吧

AlphaCPU

（当初开始写的时候起的名字真的好难听啊，写流水线的时候一定要换一个）

Verilog修BUG真的真的真的是一件十分痛苦的事啊，只能分析波形，好多问题根本没遇到过，周围没有会的人，网上资料还甚少，全靠自己抓信号分析，太痛苦了。不过确实是一件容易积累经验的事呢。

之前遇到问题大部分都解决了，记录一下吧。

1. 仿真崩溃，波形中断

可能是testbench文件本来模拟的时长就不是很长，另一方面可能是vivado配置的最大仿真时长过短，改一下配置就好了。

如果上面都不行，我想就是电路设计的问题了。

之前遇到的sll和 srl指令刚开始仿真就崩溃，当初没解决就去写别的指令了，后来发现当源操作数和目的操作数一样的时候也会产生仿真中断，一直不知道怎么回事，就去上课了，电脑放实验室，晚上上人工智能画了画，好像就发现问题了。

当初的regfile其实是仿写的别人流水线的regfile，流水线中要考虑WB阶段的数据冒险，所以会加一个判断，在同时读写的时候，将写入数据直接输出。

但是现在写的是单周期，就会发现在在回写阶段和解码阶段，如果rs == rd or rt == rd 就会产生组合电路的反馈，比如add $1,$1,4，就会导致反复加4，直到崩溃。所以将真个判断去掉就好了。 :)

2. Block Memory Generator

这个IP产生的ROM当作指令寄存器，如果PC更新和ROM读取都是上升沿的话，就会发现，PC和当前指令不对应，会晚一个周期，从而导致branch和j跳转出现问题，所以考虑将ROM的读取改为下降沿，在上升沿更新PC，更新regfile，更新MEM，在下降沿取instr。

还有就是，最后在CPU上跑了一个求和程序，但是发现不太对，最后定位问题再Block Memory Generator 生成的RAM，我发现他读地址的改变并不会导致输出数据的改变，我不太理解他这IP是怎么控制的，难道连读操作都需要clk上升沿嘛？选修课上看的，电脑没电了，就算了，直接换了Distributed Memory 比较直接一点，就没什么问题了，明天问问老师吧。

这样一个单周期CPU就写完了，实现了基本的一些指令，遇到了许多问题，最终还是全靠自己啊。

最后在CPU上跑了一个求和小程序，记录一下吧，我看GitHub上的仓库，从上周一就开始提交代码，到今天九天了，九天了啊。

杂记

心情好点了，可能是最近休息的不好吧，没事的时候看了看B站，划划水，听听歌就挺平静的。

周末QQL让我去看电影，那就去吧，出校门甚少，好像至今还没去过电影院。。

明晚可能孔来说几句，我不管了，听老师安排吧。
